## Part 2: Thematic Coding System

### How to Code Interviews

After each interview, tag themes that emerge. Use these codes for pattern recognition across interviews.

---

### Primary Codes (P Codes)

#### Problem Codes
| Code | Theme | Definition |
|------|-------|------------|
| P1 | No-shows | Patients/clients miss appointments |
| P2 | Follow-up fatigue | Too time-consuming to follow up manually |
| P3 | Payment delays | Clients don't pay on time |
| P4 | Lead leakage | Leads lost due to slow response |
| P5 | Inconsistent messaging | Different staff say different things |
| P6 | No tracking | Can't measure what's working |
| P7 | Manual errors | Human mistakes in process |
| P8 | Scalability limits | Can't grow with current process |

#### Solution Codes
| Code | Theme | Definition |
|------|-------|------------|
| S1 | WhatsApp automation | Wants automated messages |
| S2 | Payment integration | Wants Pix/payment links in chat |
| S3 | Reminders | Wants appointment reminders |
| S4 | CRM integration | Wants it connected to existing tools |
| S5 | Analytics | Wants reporting and metrics |
| S6 | Ease of use | Wants simple, no technical setup |
| S7 | Speed | Wants fast implementation |
| S8 | Cost-effective | Price-sensitive |

#### Objection Codes
| Code | Theme | Definition |
|------|-------|------------|
| O1 | Price | Too expensive |
| O2 | Complexity | Too complicated |
| O3 | Trust | Doesn't trust automation/WhatsApp |
| O4 | Integration | Won't work with existing tools |
| O5 | Time | No time to implement |
| O6 | Skepticism | Doesn't believe it will work |
| O7 | Not a priority | Other things more important |

#### Emotion Codes
| Code | Theme | Definition |
|------|-------|------------|
| E1 | Frustrated | Strong negative emotion about problem |
| E2 | Urgent | Time-sensitive, needs solution now |
| E3 | Excited | Positive reaction to solution |
| E4 | Skeptical | Doubtful, needs proof |
| E5 | Resigned | Accepts problem as unsolvable |
| E6 | Hopeful | Cautiously optimistic |

---

### Secondary Codes (S Codes)

#### Segment-Specific
| Code | Theme | Definition |
|------|-------|------------|
| SC1 | Patient no-show revenue | Lost revenue from missed appointments |
| SC2 | Staff capacity waste | Empty slots cost money |
| SC3 | Confirmation burden | Staff spends hours confirming |
| SC4 | Payment collection | Getting patients to pay is hard |
| IR1 | Lead response time | Speed critical for real estate |
| IR2 | Commission loss | Lost sales cost commissions |
| IR3 | Lead nurturing | Long sales cycle needs follow-up |
| IR4 | Trust building | Personal relationship matters |

---

### Coding Process

#### After Each Interview:
1. **Read transcript/notes**
2. **Identify themes** (use codes above)
3. **Mark frequency** (how often mentioned)
4. **Note intensity** (emotion score 1-5)
5. **Tag quotes** (link code to exact quote)

#### Example Coding:
```
INT-001 - Clínica Saúde
P1 (3x, intensity 5): "Perco 20 consultas por semana"
P3 (2x, intensity 4): "50% dos pacientes esquecem de pagar"
S1 (2x, intensity 5): "Se automatizasse, economizava 10h/semana"
O1 (1x, intensity 3): "Preciso caber no orçamento"
E1 (intensity 5): Very frustrated about no-shows
SC1 (intensity 5): "Cada no-show é R$ 200 perdido"
```

## Part 3: Pattern Recognition

### After 5+ Interviews, Look For:

#### High-Frequency Patterns
- **Problem mentioned by ≥60%** = Core problem
- **Solution wanted by ≥60%** = Must-have feature
- **Objection by ≥40%** = Major barrier to address

#### Segment Differences
- **Clinics emphasize**: P1, P3, SC1, SC2, SC3
- **Real estate emphasize**: P4, IR1, IR2, IR3

#### Correlation Patterns
- **High urgency** ↔ **High WTP**
- **Direct decision maker** ↔ **Faster close**
- **Current competitor spend** ↔ **WTP baseline**

## Part 5: ROI Calculator

### For Each Interview

#### Input Variables
| Variable | Description | Example |
|----------|-------------|---------|
| Current problem frequency | How often it happens | 50 no-shows/month |
| Cost per occurrence | Revenue lost each time | R$ 200/no-show |
| Current time spent | Hours spent on problem | 20 hours/month |
| Hourly value | Value of time (owner/manager rate) | R$ 150/hour |
| Competitor spend | Currently paying for alternatives | R$ 0/month |

#### Calculations
```
Monthly Revenue Loss = Problem frequency × Cost per occurrence
Example: 50 × R$ 200 = R$ 10,000/month

Monthly Time Waste = Time spent × Hourly value
Example: 20 hours × R$ 150 = R$ 3,000/month

Total Monthly Impact = Revenue loss + Time waste + Competitor spend
Example: R$ 10,000 + R$ 3,000 + R$ 0 = R$ 13,000/month
```

#### Output
| Metric | Value |
|--------|-------|
| Monthly Impact | R$ ___ |
| Annual Impact | R$ ___ (×12) |
| WTP | R$ ___/month |
| ROI | ___x (Impact ÷ WTP) |
| Payback Period | ___ months |

## Part 6: Go/No-Go Decision Framework

### After 15-20 Interviews, Evaluate:

#### ✅ GO Signals (3+ = Green Light)
- [ ] ≥70% recognize the problem
- [ ] ≥60% WTP ≥ R$ 300/month
- [ ] ≥50% ROI ≥ 5x
- [ ] ≥40% ready to buy in 1-3 months
- [ ] Clear pattern of problems/solutions
- [ ] Strong differentiation from competitors

#### 🟡 CAUTION Signals (2-3 = Investigate Further)
- [ ] 50-70% recognize the problem
- [ ] 40-60% WTP ≥ R$ 300/month
- [ ] Mixed ROI (3-5x)
- [ ] Fragmented patterns
- [ ] Major technical barriers

#### 🔴 STOP Signals (2+ = Pivot or Stop)
- [ ] <50% recognize the problem
- [ ] <40% WTP ≥ R$ 300/month
- [ ] <30% ROI ≥ 3x
- [ ] No clear pattern
- [ ] Strong preference for competitors
- [ ] Technical blockers
